# ラベルの版を追加する手順

小型ラベルなど、新しいラベルを作る担当者向けの説明です。
既存のコードを触る必要はありません。**HTML を1枚作って置くだけ**です。

---

## 1. 何を作るか

`screen_system/frontend/label_small.html` を作ってください。
(ファイル名は決まっています。変える場合は `config.py` の `LABEL_TEMPLATES` も直す必要があります)

一番早いのは **`label_gorilla.html` をコピーして、サイズと配置を変える**ことです。
データの取り方・バーコード・QR の描画はそのまま流用できます。

---

## 2. 守ってほしい約束(これだけ)

### ① URL パラメータ

| 呼ばれ方 | 意味 |
|---|---|
| `/label/CDI260722001LN01?print=1` | 1枚だけ表示。`print=1` があれば自動で印刷ダイアログ |
| `/label?serials=A,B,C&print=1` | 複数枚をまとめて1つのタブに表示 |

`label_gorilla.html` の末尾にこの処理が入っているので、そのままコピーすれば動きます。

### ② データの取得先

`GET /api/label/{バーコード}` を呼ぶと、次の JSON が返ります。

```json
{
  "barcodeValue": "CDI260722001LN01",
  "barcodeText":  "CDI260722001LN01",
  "fabric":       "LN",
  "width_mm":     2800,
  "height_mm":    3000,
  "product":      "1枚",
  "set_quantity": 1,
  "processing": {
    "top":    { "type": "eyelet", "spacing": 300, "skirt": false },
    "bottom": { "type": "eyelet", "spacing": 300, "skirt": true  },
    "left":   { "type": "eyelet", "spacing": 300, "skirt": false },
    "right":  { "type": "none" }
  },
  "velcroSides":     3,
  "velcroType":      "オス",
  "hasSkirt":        true,
  "skirtAttachment": "縫い付け",
  "bohenNumber":     "BOEN-2026-001",
  "qrValue":         "https://cdigolf.base.ec/"
}
```

**注意点**

- `processing.*.skirt` が `true` の辺は、ハトメとスカートが**両方**付いています。
  「ハトメ P300＋スカート」のように併記してください。
- `velcroSides` は `null`(なし) / `3`(上・左・右) / `4`(四辺) のいずれかです。
- `bohenNumber` が `"非防炎"` の時は防炎認証番号ではなく「非防炎」の意味です。

### ③ 用紙サイズ

CSS の `@page` に実寸を書いてください。ここに書いた大きさで PDF が作られます。

```css
@page { size: 110mm 200mm; margin: 0; }
```

## 3. 置いたあとにすること(정현준さん作業)

1. ファイルを `frontend/` に置いてサーバーを再起動
2. 管理画面 → **ラベル** タブを開く
   - 「未作成」だった行が「利用可」に変われば認識されています
3. その版を選んで「この版を既定にする」

### 用紙の長さが標準版と違う場合

プリンタ設定を分けます。物理的なプリンタは1台のままで大丈夫です。

**Windows で同じプリンタをもう1つ登録する**

```
設定 > プリンタとスキャナー > プリンターを追加
 > 「プリンターが一覧にない場合」
 > ローカルプリンターを手動設定
 > 既存のポートを使用 … 今使っているのと同じ USB00x を選ぶ
 > 同じドライバーを選択 > 「現在のドライバーを使う」
 > プリンター名だけ変える  例) XP-TT426B_小
```

登録できたら、その新しいプリンタの「印刷設定」で用紙の長さを小型ラベルに合わせて保存します。

**システム側に名前を教える** — `screen_system/printer_config.json`

```json
{
    "inventory_printer": "Brother QL-820NWB",
    "order_printer": "Xprinter XP-TT426B",
    "order_printer_small": "XP-TT426B_小",
    "invoice_printer": "MFC-J7100CDW Printer"
}
```

`order_printer_small` が無い場合は自動的に `order_printer` が使われます。

## 4. 参考: 標準版が二つ折りなのはなぜか

標準ラベル(110 × 410mm)は **226mm のところで折って**使います。
折り目の前後に約10mmずつ、合わせて約20mmの余白を取っています。
折って裏返すと下半分は上下逆さまに見えるので、下半分だけ 180° 回転させて印刷しています。

```html
<div class="backside">  <!-- transform: rotate(180deg) -->
  ... 洗濯表示 / QR / 会社情報 / ロゴ ...
</div>
```

折り位置には左右の端に短い線(`.foldmark`)を入れてあります。
小型版を折らないのであれば、この仕組みは不要です。
